## Copy Improvement Instructions

Review and rewrite all user-facing copy in the specified feature to be **clear, concise, and helpful** for non-technical users.

### Requirements

- **Australian English** spelling and terminology
- **Friendly but concise** - warm enough to not feel robotic, brief enough to respect users' time
- **Action-oriented** - tell users what they can do, not just what went wrong
- **Scannable** - users should understand at a glance

### What to Update

- Error messages → Structure: [What happened] + [What to do next]
- Form labels & placeholders → Remove jargon, be specific
- Headings → Clear purpose, no clever wordplay
- Instructions → Step-by-step where needed, single sentence where possible
- Buttons & CTAs → Verb-first, specific actions (e.g., "Save changes" not "OK")

### Before Making Changes

Please identify and ask about:

- **User context**: Where is the user in their journey when they see this?
- **Failure scenarios**: What specific errors might users encounter here?
- **Critical actions**: What must users understand to proceed successfully?
- **Unclear existing copy**: Any current text whose purpose isn't obvious?

_Make reasonable assumptions for minor copy, but ask about anything that could significantly impact user understanding or success._

### Apply Changes Directly

Update the copy in-place in the code. Maintain any formatting, variables, or technical requirements while improving the user-facing text.
